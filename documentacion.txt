3. Descripción inicial de la aplicación 
La aplicación será un gestor de eventos locales, que permitirá administrar eventos y los participantes asociados mediante una interfaz gráfica interactiva, desarrollada con Tkinter o PyQt.
El sistema permitirá:
Crear, modificar y eliminar eventos.


Registrar participantes y asignarlos a uno o varios eventos.


Buscar, filtrar y consultar información relevante en tiempo real.


Exportar datos en formatos CSV y PDF.


Garantizar la integridad de los datos mediante mecanismos de concurrencia, permitiendo varios usuarios conectados simultáneamente.


El almacenamiento se realizará en una base de datos MySQL, diseñada para ser clara, escalable y mantenible.

4. Justificación del proyecto 
Problema que soluciona
Muchas organizaciones locales gestionan sus eventos utilizando métodos manuales o herramientas dispersas, lo que provoca:
Duplicidad de información.


Errores frecuentes.


Dificultad para coordinar participantes.


Falta de informes automáticos.


Problemas cuando varias personas trabajan simultáneamente.





Cómo lo soluciona esta aplicación
El proyecto centraliza toda la información en una base de datos MySQL, permite el acceso simultáneo de varios usuarios y genera informes PDF/CSV de manera automática, reduciendo errores y agilizando la gestión.





5. Objetivos del proyecto 
Objetivo general
Desarrollar una aplicación de escritorio completa que permita gestionar eventos y participantes de forma centralizada, eficiente y multiusuario.
Objetivos específicos
Implementar CRUD de eventos.


Implementar CRUD de participantes.


Permitir asignación de participantes a eventos.


Gestionar la concurrencia evitando conflictos de acceso simultáneo.


Implementar exportación de datos en CSV y PDF.


Desarrollar una interfaz gráfica clara e intuitiva.


Conectar Python con MySQL usando mysql-connector o SQLAlchemy.

9.2 Fase de diseño 

Diagrama Relacional










Entidades de la base de datos
Tabla
Propósito
events
Almacena información de cada evento.
participants
Almacena datos personales de los asistentes.
event_registrations
Relación N:M entre eventos y participantes (inscripciones).
users
Gestión de usuarios de la aplicación (login, roles).
audit_logs 
Registro de acciones de los usuarios para auditoría.


Atributos de las entidades
events (Eventos)
event_id : int (PK)
title : varchar(255) NOT NULL
description : text
location : varchar(255)
start_datetime : datetime NOT NULL
end_datetime : datetime NOT NULL
capacity : int NOT NULL
status : varchar(50) NOT NULL
version : int NOT NULL (control de concurrencia)



participants (Participantes)
participant_id : int (PK)
first_name : varchar(100) NOT NULL
last_name : varchar(100) NOT NULL
email : varchar(255) NOT NULL
phone : int
identifier : varchar(50) NOT NULL (DNI/NIE)




event_registrations (Inscripciones)
registration_id : int (PK)
event_id : int (FK → events)
participant_id : int (FK → participants)
registered_at : datetime NOT NULL
status : varchar(50) NOT NULL



users (Usuarios)
user_id : int (PK)
username : varchar(50) NOT NULL
password_hash : varchar(255) NOT NULL
role : varchar(50) NOT NULL
created_at : datetime NOT NULL



audit_logs (En principio)
log_id : int (PK)
user_id : int (FK → users)
action : varchar(100) NOT NULL
entity : varchar(100) NOT NULL
entity_id : int NOT NULL
details : text
timestamp : datetime NOT NULL



Relaciones
Un evento puede tener muchas inscripciones.
Un participante puede inscribirse en muchos eventos.
Relación M:N resuelta mediante event_registrations.
users y audit_logs permiten rastrear acciones de manera opcional.
